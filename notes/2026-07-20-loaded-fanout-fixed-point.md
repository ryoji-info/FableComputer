Reply to the winning prompt of **Fable Session #2 — 2026-07-20** (Fabric 🧵, 2-of-3). Labels: **demonstrated** (cite the run/key), **in-model** (name the model + assumptions), **open**. A well-argued "the honest fan-out is only 2" is the prompt's stated success condition — and it is what the numbers give.

## 1. Verdict

**The honest, margin-reserved fan-out a single gain cell can drive is F = 2. The *raw* regeneration ceiling is F = 4 (near-specular contact) / F = 3 (pessimistic contact) — higher than the naïve fixed-gain ledger's ~3.2 because sub-knee self-compensation genuinely lifts the per-cell gain — but reserving a real (~2 dB) regeneration margin and holding the k = 8 static noise margin pulls the datapath-quotable number down to 2.** The two corrections the prompt flagged act in opposite directions and *both matter*: the F-way split softens each branch's drive by −10·log₁₀F dB, which (a) **raises** the per-cell gain along the cascade curve (CW knee-anchored: 7.63 dB at F=1 → 8.52 at F=2 → 9.06 at F=4, as compression relaxes — demonstrated, §5) yet (b) **lowers** the effective input sharpness k_eff = 8·F^(−½) (8.0 → 5.66 → 4.0) and the absolute static noise margin with it. Raw regeneration (net loop gain ≥ 1) rides the rising gain to F = 4; but at F = 4 the reserve is only +0.48 dB (near-specular) and *negative* at the pessimistic contact corner, and k_eff has fallen to 4.0 (50 % of design). The largest F that keeps a ~2 dB regeneration reserve across **both** contact corners and **both** CW/pulse bands, while retaining ~86 % of the design static noise margin (k_eff = 5.66, NM_frac 0.221 vs 0.256), is **F = 2**. **This selects the Part III RFC fan-out row: quote F = 2 (margin-reserved), with F = 3 reachable only if the contact is first shown near-specular and a thin ≤2 dB reserve is accepted, and F = 4 the raw ceiling with no reserve.** It is a genuinely composed-logic result the half-adder never exercised, and it consumes the #47 boundary gate as a propagated ±1 dB corner rather than reopening it.

## 2. The loaded fan-out ledger (fixed-point, demonstrated)

Model (in-model, all stated): equal *power* split into F branches ⟹ each branch drive is A_op·F^(−½) in amplitude (−10·log₁₀F dB); per-cell gain = G_CW − compression(drive), G_CW = 9.661 dB (small-signal CW base, demonstrated), compression = 1/√(1+(drive/A_SAT)²) with **A_SAT recalibrated to the measured knee** (A_SAT = 0.0150 CW / 0.0226 pulse, [notes/2026-07-17-drive-sweep-knee-anchored.md](https://github.com/ryoji-info/FableComputer/blob/main/notes/2026-07-17-drive-sweep-knee-anchored.md) §5) so F=1 reproduces the knee-anchored band **+7.63 CW / +8.64 pulse** (demonstrated, §5); per-gate loss 2.555 dB (353 K, `per_gate_loss_353K_dB`) paid per branch; **regeneration margin = gain(F) − 10·log₁₀F − per_gate_loss**; effective sharpness k_eff = 8·F^(−½); NM_frac = ½(1−2/k_eff)(1−10^(−10/20)). The fixed point: the drive each load receives sets its operating point → its gain → the swing it restores and re-presents. Ledgers (demonstrated, §5):

## 3. The two F_max, and why they differ

- **Raw F_max (regeneration only, margin ≥ 0): 4** (near-specular) / **3** (pessimistic). This is *higher* than the naïve fixed-gain ledger F = 10^((gain−loss)/10) = 10^((7.63−2.56)/10) ≈ 3.2, because the gain is not fixed — the split drives each cell below the 1-dB knee where the restoring rail gives back compression (in-model self-compensation; demonstrated gain rise 7.63→9.06 dB, §5).
- **Margin-reserved F_max (the honest datapath number): 2.** It requires a ~2 dB regeneration reserve (against drift, temperature, aging, and the ±1 dB contact uncertainty) *and* retention of a k = 8-class static noise margin. F = 2 holds +2.96 dB (near-specular CW) / +1.96 dB (pessimistic CW) / +3.56/+2.56 dB (pulse) and k_eff = 5.66 (86 % of design NM). F = 3 drops the pessimistic-CW reserve to +0.55 dB.
- **Why they differ:** raw regeneration only asks "does the loop clear unity?", which self-compensation carries to 4; the honest number additionally reserves margin and preserves discrimination sharpness, which the split necessarily erodes (k_eff = 8·F^(−½)). No fan-out preserves the *full* k = 8 margin — any split softens the drive — so the design must accept reduced input sharpness; F = 2 accepts the least (14 % NM loss) while keeping a real reserve.

**ISI (in-model, demonstrated):** the released `cell.isi_spread_dB` gain-recovery spread is **0.00 dB at 0.25 THz, 0.02 dB at 1 THz** — negligible at the operating symbol rate, so ISI neither tightens nor sets the fan-out at F = 2 (it would matter only near the raw ceiling at high symbol rate; second-order here).

## 5. Runnable listing

Self-contained apart from importing the released `fable-model-chain`.

```python
# -*- coding: utf-8 -*-
# Loaded fan-out fixed-point ledger — Fable Session #2, 2026-07-20 (Opus 4.8, disclosed).
# Reproduces G_CW, cascade_per_cell_dB, the knee-anchored band, and the fan-out ledger.
import math, os, sys, json
CHAIN = r"...\fable-model-chain"                      # adjust to your checkout
sys.path.insert(0, CHAIN)
import constants as C, regen as R, cell as CELL
RES = json.load(open(os.path.join(CHAIN, "results.json"), encoding="utf-8"))
G = R.cw_net_gain_dB(0.7)                 # 9.661 small-signal CW base
A_op = 0.0116; PGL = RES["per_gate_loss_353K_dB"]      # 2.555
comp = lambda s, asat: 20*math.log10(1/math.sqrt(1+(s/asat)**2))
gain = lambda F, asat: G + comp(A_op*F**-0.5, asat)   # knee-anchored per-cell gain at F-split
nm   = lambda k: 0.5*(1-2.0/k)*(1-10**(-0.5))
# (A) anchors
print("G_CW=%.6f (json %.6f); cascade[0]=%.4f (json %.4f); NM(8)=%.4f (json %.4f)"
      % (G, RES["cw_regen_gain_dB_at_0p7"], gain(1,0.02), RES["cascade_per_cell_dB"]["0"],
         CELL.static_noise_margin(), RES["noise_margin_frac"]))
print("knee-anchored F=1: CW=%.2f (note 7.63), pulse=%.2f (note 8.64)"
      % (gain(1,0.0150), gain(1,0.0226)))
# (C) ledger
for asat,band in ((0.0150,"CW"),(0.0226,"pulse")):
  for corner,cl in ((0.0,"near-spec"),(-1.0,"pessim")):
    row=[]
    for F in (1,2,3,4,5):
        m = gain(F,asat)+corner - 10*math.log10(F) - PGL
        row.append((F, round(gain(F,asat)+corner,2), round(m,2), round(8*F**-0.5,2), round(nm(8*F**-0.5),3)))
    raw=max(F for F,_,m,_,_ in row if m>=0)
    res=max((F for F,_,m,_,_ in row if m>=2.0), default=1)
    print(f"{band:5s} {cl:9s}: raw F_max={raw}, reserved(>=2dB) F_max={res} | "
          + " ".join(f"F{F}:{m:+.2f}dB/k{k}" for F,_,m,k,_ in row))
```

## 6. Limitations

1. **Executing model.** Opus 4.8, not Fable 5 (header).
2. **In-model idealizations.** Equal, lossless power split (a real splitter/combiner adds insertion loss that *lowers* F_max); the tanh transfer and the compression law 1/√(1+(swing/A_SAT)²) of `cell.py`; A_SAT recalibrated to the measured knee ends (±0.5 dB gain class per 07-17); k_eff = 8·F^(−½) assumes the input logic swing scales with branch amplitude and dn fixed — defensible but in-model. The 2 dB reserve threshold is an engineering choice, not a derived floor; the ledger is given so a different reserve can be read off directly.
3. **Contact gate inherited, not resolved.** The ±1 dB corner is the 07-18 allowance; #47 left the exponent p open and leaning pessimistic (p=1 ⟹ ΔG ≈ −5.7 dB ⟹ F=1). This fan-out is only as trustworthy as that gate.
4. **Single operating point / temperature (353 K CW + pulse).** No loaded-cascade time-domain simulation was run — this is the fixed-point *ledger*; a full multi-cell loaded transient (with a real splitter, pattern-dependent ISI at the softened operating point, and disorder) is the WP1 check this predicts.
5. **Everything rides on the unproven gain cell** (bench gate G1).
